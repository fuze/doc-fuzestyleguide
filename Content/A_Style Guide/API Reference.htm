Terms-API 

(Submit comments here)

An important aspect of API usability is consistency. To provide our external API consumers with a better experience, please refer to the following terminology when working with or writing about publicly facing Fuze APIs.

Term Definition
Organization This represents a legal entity that has an association with Fuze. Customers are a type of organization. This used to be called customer.
Portal Legacy Account A legacy construct in the Portal that was used for billing purposes
NetSuite Billing Account A construct which groups information for billing purposes.
Billing Location The location where a customer's invoice is sent.
Service An instance of sku/product that is provisioned based on the feature set of the SKU at the time of provisioning
ServiceID Internal immutable idenfitier of the service.
Service Location Code The alpha numeric string (usually human readable) identifying where the service is located.
ServiceLocationID Internal immutable idenfitier where a service is located.
Department A construct used to group users and services where each user / service may only be part of one department.
Department Code The alpha numeric string (usually human readable) identifying the deparment.
DepartmentID Internal immutable idenfitier of a department.
User The representation of the end user (person) who is part of an organzation.
Username A unique (and changable) identifier used by users to log into the Fuze platform and to identify this user externally.
UserID Internal immutable idenfitier of a user.
Resource A resource is a thing that a service can be granted to: a room, a desk, a person. Users are a type of resource.
PBX A pbx is a name-space, used to isolate extensions; either for design, geographic or scalability reasons.
Carrier A company that provides us telephony services or connections within a country or region.
Phone Number A phone number (used to be called DID).
